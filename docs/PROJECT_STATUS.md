Genuinely new engineering territory beyond the two attribute modules: a hook handler registered with both an invoke *and* a compensate function (not just invoke), so a later cart-completion step's failure automatically releases the capacity this hook booked — verified with real, Redis-backed concurrency tests (two-of-one, three-of-eight, and release-then-rebook scenarios), not mocked.
